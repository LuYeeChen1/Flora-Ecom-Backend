# 后端功能文档编写流程

用于更新或新增后端功能说明文档的操作流程。

## 步骤
1. 明确要描述的功能范围（接口、认证、数据存储、异常处理等）。
2. 从代码定位入口与关键流程：
   - 接口层（Controller）
   - 应用层（Service / UseCase）
   - 基础设施层（Repository / Client / Config）
   - 领域层（Model / Error）
3. 逐一列出功能步骤：
   - 请求入口
   - 业务处理
   - 外部依赖（数据库 / Cognito）
   - 响应输出
4. 为每个功能列出对应文件清单（路径 + 关键类名）。
5. 复核配置文件（`application*.properties`）中的必要配置项并标注。
6. 完成后将文档放入 `docs/`，并在提交说明中注明新增文档。
